Progress Tracker Approaches

Choose the structure that best fits your workflow

A

Centralized TRACKER.md

A single, high-visibility file at the root. Uses GFM checkboxes for easy tracking and git history review.

  • Easy to read/edit
  • Version controlled
  • Zero dependencies
B

Docs Progress Folder

Separate files for each milestone in docs/progress/. Great for complex, long-term projects.

  • Avoids file bloat
  • Granular history
  • More navigation needed
C

JSON Manifest

Structured progress.json. Best if you plan to build custom scripts or dashboards.

  • Harder to read manually
  • Overkill for simple tasks
